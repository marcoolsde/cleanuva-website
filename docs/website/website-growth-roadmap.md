# Cleanuva website — growth roadmap

Short living note for SEO/attribution work that is deliberately deferred. Keep it
lean; expand only when a phase is actually picked up.

## Current state (P13A — done)

Search-engine indexing readiness only. No analytics, no ad tracking, no cookie
banner, no third-party scripts.

- `sitemap.xml` — generated by `src/app/sitemap.ts` (core pages + legal pages at
  low priority), locale-prefixed, with hreflang alternates for en/de/zh/ar.
- `robots.txt` — generated by `src/app/robots.ts`, points to
  `https://cleanuva.ai/sitemap.xml`, disallows internal paths only.
- `metadataBase = https://cleanuva.ai`; per-page canonical + hreflang via
  `src/lib/seo.ts` (`pageMetadata`).
- No `noindex` on public pages (only the internal `/kitchen-sink` dev page is
  `noindex` and is excluded from the sitemap).

**Search Console / Bing verification:** prefer **DNS TXT (Domain property)** — no
code change, covers all paths, no cookie/privacy impact. Bing can *Import from
Google Search Console*. Do NOT hardcode fabricated tokens.
- If HTML-file verification is ever required: drop the provided file into
  `public/` (served at the site root).
- If meta-tag verification is ever required: add `verification` to the metadata in
  `src/app/[locale]/layout.tsx` (Next supports `verification.google` and
  `verification.other["msvalidate.01"]`) — only once a real token is provided.

## P13B — LinkedIn and analytics tracking roadmap

### P13B-1 — Lightweight LinkedIn attribution without third-party tracking (do first)

Attribution with **no** GA4, **no** LinkedIn Insight Tag, **no** cookie banner —
so it stays consent-free and legally clean.

- Define a LinkedIn UTM naming convention (source / medium / campaign / content / term).
- Forms auto-capture `utm_source`, `utm_medium`, `utm_campaign`, `utm_content`, `utm_term`.
- Forms auto-capture `landing_page`, `current_page`, `referrer`.
- Surface this source info inside the n8n lead email.
- No GA4. No LinkedIn Insight Tag. No cookie banner needed.
- Best fit: organic LinkedIn posts, post-exhibition private follow-up, links
  customers forward.

### P13B-2 — Paid ads tracking with consent (only when running paid ads)

Trigger this only when actually spending on LinkedIn Ads / Google Ads.

- Cookie consent banner.
- Consent Mode / consent-aware script loading.
- LinkedIn Insight Tag.
- LinkedIn conversion events: `request_demo_submit`, `get_pricing_submit`,
  `accessories_inquiry_submit`.
- Optional GA4.
- Update Privacy Policy and Cookie Policy (Terms need no major change).
- Load non-essential tracking **only after** user consent — never load
  advertising/analytics cookies without consent.

### Decision principle

Do **not** do P13B-2 now: the site currently has no non-essential cookies, which
keeps legal and load cleaner. For the next LinkedIn push, prefer P13B-1 (UTM +
form source capture) first.
